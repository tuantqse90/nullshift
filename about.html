Core Philosophy

// the principles we build on

privacy.md

"Privacy is not a feature"

It's a fundamental right. Every tool we build starts with the question: how do we protect the user? If the answer isn't "by default," we start over.

open-source.md

"Open source or bust"

Transparency is trust. Our code is public, auditable, and free. No black boxes. No proprietary lock-in. If you can't read the source, you can't trust the system.

threat-model.md

"Build for the worst case"

Assume the network is hostile. Assume the server is compromised. Design for adversarial conditions. If it survives the worst case, it's ready for production.
